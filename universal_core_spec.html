R359 — Universal Core Spec

Every business runs on the same 80-form skeleton.

"It needs to be doable for any type of business." — universal core + industry overlay. Same Murphy pipeline, any tenant.
The pattern. Every business — MEP contractor, accounting firm, restaurant, healthcare practice, SaaS company, retailer — has the same 80-form operational skeleton across People / Money In / Money Out / Compliance / Records / Risk. The vertical-specific layer sits ON TOP of this, never replaces it. R358 proved depth on one industry; R359 proves the pattern is universal.

The Architecture

TENANT_PROFILE (identity + brand + scale + industry) │ ├──── UNIVERSAL CORE (~80 forms — applies to all) │ F-CORE-1 People (HR + payroll + benefits) │ F-CORE-2 Money In (sales + AR) │ F-CORE-3 Money Out (AP + expenses) │ F-CORE-4 Compliance (entity + tax + registrations) │ F-CORE-5 Records (governance + board + audit) │ F-CORE-6 Risk (insurance + contracts) │ └──── INDUSTRY OVERLAY (15–105 forms — vertical-specific) MEP → DOB filings, SST, prevailing wage, AIA Accounting → engagement letters, peer review, 7216 Restaurant → DOH grade, food protection, liquor Healthcare → HIPAA, BAA, NPI, credentialing SaaS → SOC2, MSA, DPA, sub-processor list Retail → sales tax per state, PCI, return policy

The same Murphy pipeline assembles a tenant's full form pack by joining the universal core to the relevant overlay. Brand assets (logo, colors, fonts, voice) are tenant-specific and override default styling at render time.

Decision & Owner

Decision: Treat the 80-form universal core as the foundation for every Murphy tenant. A new tenant onboarding flow always instantiates the universal core, then layers the industry overlay matching their NAICS / Murphy domain category. Brand is the tenant-specific veneer.

What R359 proves. Two new tenants — Rosewood CPA Partners (accounting) and Bedford Stuyvesant Eats (restaurant) — were spun up using the same pattern. Both render the same universal core with their own brand and industry overlay. See the demo links above.
