What Murphy Does

Every Operation. One System.

Murphy is not a chatbot. It's an AI operating system — 1,661 Python modules running every department of your business on one substrate. Every claim below links to live data you can verify yourself.

🧠
CEO Console
Talk to Murphy in plain English via /api/chat. It routes through the unified voice (murphy_voice.py), runs an audit snapshot, and replies with grounded context. A scheduled morning_brief job runs every day via APScheduler. Voice channel wired via Twilio (text-to-voice and PSTN dial-back ready, TTS engine wiring in progress).
Text Live · Voice In Build
⚡
Autonomous Swarm
Real agent files in src/: collector_agent, executor_agent, auditor_agent, exec_admin_agent, prod_ops_agent, agentic_comms_router, agent_email_chain, agentic_onboarding_engine, plus role-souls dispatched via rosetta_org_router. Tasks flow through the MFGC 7-phase gate pipeline; everything is auditable end-to-end at /mission.
8 Agent Modules Wired
📁
File Intelligence
Living-document endpoints live today: /api/documents/{id}/magnify, /simplify, /solidify, /gates. Causality sandbox modules: causality_commission.py, causality_sandbox.py, causal_spike_analyzer.py. PDF/DOCX/CSV ingestion handlers in build via the same MFGC gate path — every artifact gets compliance + provenance scoring before it lands.
Causality Sandbox Live
🛡️
Compliance Engine
5 frameworks wired into compliance_engine.py today as ComplianceFramework enums: HIPAA, SOC2, GDPR, ISO27001, PCI-DSS. SOX, CCPA, NIST-CSF documented but not yet enum-mapped. Per-tenant toggles via compliance_toggle_manager. Industry presets auto-apply (healthcare → HIPAA+SOC2, finance → PCI-DSS+SOC2+ISO27001, eu_data → GDPR+ISO27001). MurphyCritic gate enforces at build time, not runtime.
5 Frameworks Live · 3 In Build

CRM + Outreach
239 contacts in /var/lib/murphy-production/crm.db: 43 classified as leads or prospects, plus business-directory entries waiting to qualify. Murphy drafts cold outreach, routes through the HITL review queue (R384), and only releases to the wire after founder approval. Outbound mail signed with DKIM at mail._domainkey.murphy.systems via OpenDKIM, relayed through this server's Postfix.
239 Contacts Live

Treasury & Cost Ledger
Five ledgers backing money flow: treasury.db (operations_wallet, journal_entries, payment_records), billing.db (subscriptions, IPN events, NOWPayments plans), llm_cost_ledger.db (every LLM call with tenant_id since R405), capital_engine.db (729 spend proposals tracked), economic_pulse.db. Daily P&L rollup at tenant_daily_pnl. Pre-revenue today — current monthly LLM burn projects ~$37 from telemetry. Crypto revenue split is aspirational, not live.
5 Ledgers · Per-Tenant P&L
🎤
Voice Control
Mercy announcer in murphy_voice.py generates narration scripts from run summaries (title, confidence, HITL count, outcome). Voice bridge live (voice_bridge.py). Twilio phone integration: outbound dial path proven, inbound webhook ready. STT, speaker recognition, and PiCar-X embodiment are roadmap items, not yet shipped.
Announcer Live · STT Roadmap
🔗
Multi-Provider LLM
Live 3-layer cascade verified in R398: DeepInfra Qwen3-235B (primary) → Together.ai Llama-3.3-70B-Instruct-Turbo (fallback) → local Ollama phi3:latest (failsafe). Every call captured in llm_cost_ledger.calls with tenant_id, tokens, latency, cost (R405). Bring-your-own-OpenAI-key on roadmap — codebase already uses the OpenAI SDK protocol so wiring is shallow.
3 Providers · Per-Tenant Token Ledger
