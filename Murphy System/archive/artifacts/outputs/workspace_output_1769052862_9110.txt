## Understanding the Complete System

### The Big Picture: From Setup to Operation

The Murphy System is NOT just a terminal or UI - it's a **complete business automation lifecycle** with 4 distinct phases:

## Phase 1: System Setup (Swarm-Based Initialization)

**Purpose:** Discover what the system needs to do through intelligent questioning

**How It Works:**
1. User provides initial context (company type, problem, or lets Librarian guide)
2. System spawns swarms to ask discovery questions
3. Questions cover: domains, constraints, goals, resources, risks
4. Answers feed into constraint generation
5. System builds organizational structure
6. Gates are synthesized based on constraints

**Key Components:**
- Swarm-based question generation (Creative, Analytical, Hybrid)
- Domain coverage analysis (Physical, Agentic, Hybrid, Shadow)
- Constraint discovery (Budget, Regulatory, Architectural, etc.)
- Organizational chart generation
- Gate synthesis

## Phase 2: Operation (Terminal/Librarian/Plan Review/Full UI)

**Purpose:** Day-to-day interaction and task execution

**Four Interaction Modes:**

### A. Terminal Interface
- Command-driven interaction
- `/help`, `/status`, `/initialize`, etc.
- Direct control over system operations
- Command history and autocomplete

### B. Librarian System
- Intent mapping and natural language understanding
- Transcript archive management
- System knowledge base
- Learning from interactions

### C. Plan Review Interface
- Living documents that evolve
- Buttons: Accept, Edit, Reject, Magnify, Simplify, Solidify
- Human-in-the-loop approval
- Document lifecycle management

### D. Full UI
- Visual dashboards and graphs
- Agent network visualization
- State tree exploration
- Process flow monitoring
- Real-time metrics

## Phase 3: Swarm Execution

**Purpose:** Parallel processing of tasks with different approaches

**Six Swarm Types:**
1. **CREATIVE** - Divergent thinking, exploration
2. **ANALYTICAL** - Systematic, data-driven
3. **HYBRID** - Combined creative and analytical
4. **ADVERSARIAL** - Challenge assumptions
5. **SYNTHESIS** - Merge perspectives
6. **OPTIMIZATION** - Refine solutions

**Execution Flow:**
1. Task divided into swarm-appropriate subtasks
2. Multiple swarms work in parallel
3. Results validated through gates
4. Artifacts generated and stored
5. Learning captured for future use

## Phase 4: Continuous Learning

**Purpose:** System improves over time through observation

**Shadow Agent System:**
- Observes human actions and decisions
- Identifies patterns and preferences
- Proposes automation opportunities
- Learns domain-specific knowledge
- Adapts to organizational culture

**AI Director:**
- Monitors all system interactions
- Detects errors and inefficiencies
- Escalates to supervisors when needed
- Recommends process improvements
- Tracks system evolution

## System Architecture Summary

```
Frontend (Port 3000) ←→ Backend (Port 3000) ←→ LLMs (Groq, Aristotle, Onboard)
```

### 8 Component Types:
1. **Agents** - Autonomous entities executing tasks
2. **Components** - Modular building blocks
3. **Gates** - Validation checkpoints
4. **Systems** - Complete operational workflows
5. **States** - Snapshot conditions with parent-child relationships
6. **Deterministic Systems** - Predictable processes
7. **Verified Systems** - Systems that passed validation
8. **Generated Systems** - Dynamically created by Murphy
