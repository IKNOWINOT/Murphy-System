New error code registered: "sqlite3.Row treated as dict — use dict() cast."

Honest finding: the SQL UPDATE for status change committed before the dict construction crashed. So in the failed run, status DID flip to 'departed' but the JSON response failed to print. I had to rollback both departures via SQL and re-run cleanly to get clean proof output. This is a R355 catch about boundary discipline — never commit DB state before the entire response path is exercised.
